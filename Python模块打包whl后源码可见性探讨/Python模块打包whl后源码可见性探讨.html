Python模块打包成whl文件后的源码可见性

当你将Python模块打包成whl（wheel）文件后，源码的可见性取决于你如何打包：

默认情况

  1. 纯Python包：如果你只是简单打包Python代码（.py文件），这些文件会原样包含在whl中，安装后用户可以直接看到源码。

  2. 编译扩展：如果是C扩展等编译代码，会包含编译后的二进制文件（如.so或.pyd），而不是源码。

保护源码的方法

如果你想保护源码不被轻易查看，有几种方法：

  1. 编译为字节码：

    • 使用compileall将.py文件编译为.pyc字节码文件

    • 然后只打包.pyc文件而不包含.py文件

  2. 使用Cython：

    • 将Python代码转换为C代码并编译

    • 这会生成二进制扩展模块

  3. 代码混淆工具：

    • 使用如pyobfuscate等工具混淆代码

    • 注意这不能完全防止反编译，只是增加难度

示例：只包含.pyc文件

在setup.py中：

python
Copy
Download
from setuptools import setup
import compileall
import os

# 先编译所有.py文件
compileall.compile_dir('your_package', force=True)

# 删除.py文件，只保留.pyc
for root, dirs, files in os.walk('your_package'):
    for file in files:
        if file.endswith('.py') and not file == '__init__.py':
            os.remove(os.path.join(root, file))

setup(
    name='your_package',
    packages=['your_package'],
    package_data={'your_package': ['*.pyc']},
    # 其他参数...
)

注意事项

  • 完全保护Python源码非常困难，因为.pyc文件也可以被反编译

  • 对于真正需要保护的代码，考虑使用C/C++编写核心部分

  • 开源许可证可能要求提供源码

最安全的保护方法是将核心算法放在服务端，客户端只通过API调用。

解决打包后模块导入错误问题

你遇到的 ModuleNotFoundError: No module named 'secretmodule.core' 错误是因为在打包过程中处理 .pyc 文件的方式不正确。下面我提供一个修正后的完整解决方案。

修正后的完整打包流程

1. 项目结构

Copy
Download
my_secret_module/
├── setup.py
└── src/
    └── secretmodule/
        ├── __init__.py
        ├── core.py
        └── utils.py

2. 修正后的 setup.py

python
Copy
Download
import os
import compileall
import shutil
from setuptools import setup, find_packages

# 临时构建目录
BUILD_DIR = "build_temp"
SRC_DIR = "src"

# 清理并创建临时目录
if os.path.exists(BUILD_DIR):
    shutil.rmtree(BUILD_DIR)
os.makedirs(BUILD_DIR)

# 复制源代码到临时目录
shutil.copytree(os.path.join(SRC_DIR, "secretmodule"), 
                os.path.join(BUILD_DIR, "secretmodule"))

# 编译所有Python文件为.pyc
compileall.compile_dir(BUILD_DIR, force=True, legacy=True)

# 处理.py和.pyc文件
for root, dirs, files in os.walk(BUILD_DIR):
    for file in files:
        full_path = os.path.join(root, file)
        
        # 删除.py文件（保留__init__.py）
        if file.endswith('.py') and file != '__init__.py':
            os.remove(full_path)
        
        # 将.pyc文件移动到正确位置并重命名
        if file.endswith('.pyc'):
            # 从类似 '__pycache__/core.cpython-38.pyc' 提取模块名
            if os.path.basename(root) == '__pycache__':
                module_name = file.split('.')[0]
                new_path = os.path.join(os.path.dirname(root), f"{module_name}.pyc")
                os.rename(full_path, new_path)

# 确保__init__.py存在
open(os.path.join(BUILD_DIR, "secretmodule", "__init__.py"), 'a').close()

setup(
    name="my_secret_module",
    version="0.1",
    packages=find_packages(where=BUILD_DIR),
    package_dir={"": BUILD_DIR},
    package_data={"secretmodule": ["*.pyc"]},
    include_package_data=True,
    description="A module with protected source code",
    author="Your Name",
    author_email="your.email@example.com",
)

3. 关键修正点说明

  1. 添加了 legacy=True 参数：确保生成旧式 .pyc 文件格式

  2. 正确处理 __pycache__ 目录：

    • 将 .pyc 文件从 __pycache__ 移动到模块目录

    • 重命名为正确的模块名（如 core.pyc 而不是 core.cpython-38.pyc）

  3. 添加了 package_data 和 include_package_data：确保 .pyc 文件被包含在包中

  4. 使用 src 目录结构：更标准的Python项目布局

from setuptools import setup, find_packages
from Cython.Build import cythonize
from Cython.Distutils import build_ext
import os

# 删除所有.py文件（保留__init__.py）
def remove_py_files(root_path):
    for root, dirs, files in os.walk(root_path):
        for file in files:
            if file.endswith('.py') and file != '__init__.py':
                os.remove(os.path.join(root, file))

# 先编译再删除.py文件
class CustomBuildExt(build_ext):
    def run(self):
        super().run()
        for ext in self.extensions:
            if ext.sources[0].endswith('.py'):
                remove_py_files(os.path.dirname(ext.sources[0]))

setup(
    name="my_protected_module",
    version="0.1",
    packages=find_packages(where="src"),
    package_dir={"": "src"},
    
    # 使用Cython编译所有.py文件（除了__init__.py）
    ext_modules=cythonize(
        [
            "src/secretmodule/core.py",
            "src/secretmodule/utils.py"
        ],
        compiler_directives={'language_level': "3"},
        build_dir="build/cython"
    ),
    
    cmdclass={'build_ext': CustomBuildExt},
    zip_safe=False,
    
    # 确保包含非Python文件
    include_package_data=True,
    
    # 其他元数据
    author="Your Name",
    author_email="your.email@example.com",
    description="Module with Cython-protected source code"
)
